# Purpose

This document establishes the official engineering vocabulary of Aura-Nova Tech Studio.

Its purpose is to ensure that every engineer, contributor, AI system, partner, and stakeholder uses consistent terminology across all products, documentation, repositories, and communications.

This lexicon serves as the authoritative reference for engineering language.

---

# Core Principles

Language creates clarity.

Clarity enables engineering.

Engineering builds products.

Products create value.

# Company Terms

## Aura-Nova

Aura-Nova Tech Studio is an AI-native venture studio focused on building secure, intelligent software platforms for Africa and global markets.

---

## Digital Headquarters

The Digital Headquarters is the permanent operational environment of Aura-Nova.

It consists of:

- GitHub (Engineering Source of Truth)
- Google Drive (Corporate Knowledge Library)

Together these systems contain the company's architecture, governance, documentation, engineering assets, and operational knowledge.

---

## Operation Genesis

Operation Genesis is the foundational programme responsible for establishing the governance, engineering standards, architecture, culture, and operating principles of Aura-Nova.

---

## Project Atlas

Project Atlas is the engineering execution programme responsible for designing, building, testing, deploying, and maintaining Aura-Nova platforms and products.

---

# Platform Terms

## HELIX

HELIX is the AI-native technology platform developed by Aura-Nova.

It provides the shared services, infrastructure, intelligence, security, and operational capabilities used across all Aura-Nova products.

---

## HELIX OS

HELIX OS is the core operating platform.

It provides:

- Authentication
- Authorization
- Configuration
- API Gateway
- Events
- Logging
- Notifications
- User Management

HELIX OS contains platform capabilities only.

---

## HELIX Core

HELIX Core contains reusable engineering components shared across every product.

Examples include:

- Shared APIs
- Common libraries
- Core services
- Integration components

---

## HELIX AI

HELIX AI is the intelligence layer of the platform.

Responsibilities include:

- AI routing
- Prompt management
- Knowledge retrieval
- Memory
- Model orchestration
- Evaluation

---

## HELIX Cyber

HELIX Cyber is Aura-Nova's cybersecurity platform.

Its mission is to provide enterprise-grade cybersecurity operations, monitoring, detection, response, and compliance for organizations of all sizes.

# Governance Terms

## Genesis Directive (GD)

A Genesis Directive defines strategic decisions governing the organization.

Example:

GD-000

GD-001

GD-002

---

## Genesis Engineering Standard (GES)

A Genesis Engineering Standard defines mandatory engineering practices.

Example:

GES-001

---

## Genesis Product Specification (GPS)

A Genesis Product Specification defines functional and technical requirements for products.

Example:

GPS-001

---

## Architecture Decision Record (ADR)

An ADR documents significant engineering decisions, including context, alternatives considered, final decisions, and consequences.

# Engineering Maturity Levels

Every repository and product progresses through the following maturity model.

| Level | Name | Meaning |
|--------|------|---------|
| L0 | Vision | Concept and opportunity identified |
| L1 | Defined | Requirements and governance documented |
| L2 | Structured | Repository and engineering standards established |
| L3 | Engineered | Active implementation underway |
| L4 | Operational | Production-ready or deployed |
| L5 | Optimized | Continuous improvement through automation and measurement |
